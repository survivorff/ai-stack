# 🔍 市场调研

> 创业者 / PM / 营销人最高频的"重脑力"任务。**AI 让"1-2 周调研"变成"1-2 天"**。

---

## 你为什么读到这里?(痛点)

- **信息碎片化**：行业数据散在 10+ 个源（报告 / 新闻 / 论坛 / 社交）
- **深度调研慢**：手工搜 + 读 + 整理 + 写结论 = 1-2 周
- **国内外资料散在不同平台**：中文在知乎 / 公众号，英文在 Twitter / 论文
- **数据可靠性难判断**：哪些数据是真的？哪些是营销吹的？
- **调研完不知道怎么得出结论**：看了 100 篇但"so what?"

---

## 推荐方案

### 🥇 方案 A：Gemini Deep Research（海外深度调研）

**适合**：海外市场 / 英文资料为主 / 要出正式报告
**时间**：1-2 小时（含 AI 自动跑 30-60 分钟）
**工具**：[Gemini 3 Pro Deep Research](../tools/foundation-models/gemini.md)
**预算**：$20/月

**步骤**：
1. 描述调研主题 + 你想搞清楚的 3-5 个具体问题
2. Gemini 给你"研究计划" → 你确认 / 改
3. AI 跑 30-60 分钟（读 50+ 网页，做笔记，交叉对比）
4. 输出 5000-15000 字带引用的综述
5. 你 review + 提炼结论

**杀手锏**：AI 自动读 50+ 网页 + 交叉对比 + 给引用源。**人工做这件事 = 1-2 周**。

### 🥈 方案 B：Kimi 探索版 + 秘塔（国内中文调研）

**适合**：国内市场 / 中文资料为主
**时间**：1-3 小时
**工具**：[Kimi 探索版](../tools/foundation-models/kimi.md) + [秘塔](../tools/research/mita.md)
**预算**：¥0

**步骤**：
1. 秘塔搜"行业 + 关键词"→ 快速出 5-10 条核心信息
2. Kimi 探索版追问 5-10 轮（深入挖）
3. 把关键资料上传 Kimi 长上下文 → 让它做交叉对比
4. 让 Claude / DeepSeek 写结论

### 🥉 方案 C：Perplexity Pro Search（快速搜索式）

**适合**：已经知道在找什么，只想快速结果
**时间**：30 分钟
**工具**：[Perplexity](../tools/research/perplexity.md)

**步骤**：
1. Perplexity Pro Search 问具体问题
2. 看引用源 → 判断可靠性
3. 追问 3-5 轮深入

## 关键 Prompt 模板

### Prompt 1：完整市场调研（给 Gemini Deep Research）

```
我要做关于 [行业 / 赛道 / 产品] 的市场调研。

具体想搞清楚：
1. 市场规模（TAM / SAM / SOM，最近 3 年数据）
2. 主要玩家（头部 5-10 家，各自定位 / 份额 / 融资）
3. 用户画像（谁在用 / 为什么用 / 付多少钱）
4. 商业模式（怎么赚钱 / 单位经济模型）
5. 趋势（过去 2 年变化 + 未来 1-2 年预测）
6. 机会 / 威胁（哪里有空白 / 哪里有风险）

预期输出：
- 5000-10000 字综述
- 核心结论先行（TL;DR 200 字）
- 每个论断有引用源
- 标出"数据可靠度"（官方 / 第三方 / 推测）
- 最后给"行动建议"（如果我要进入这个市场）

读者：[创业者 / PM / 投资人 / 自己]
```

### Prompt 2：竞品对比矩阵

```
我要对比 [竞品 A / B / C / D]。

帮我做对比矩阵：

| 维度 | A | B | C | D |
|------|---|---|---|---|

维度（请基于这些扩展）：
- 核心功能
- 定价
- 目标用户
- 技术栈（如果公开）
- 融资 / 团队规模
- 国内可用性
- 优势 / 劣势
- 最近 6 个月的关键动作

每格简短 + 引用源。
最后给：在我场景下的推荐 + 1 句话理由。
我场景：[描述]。
```

### Prompt 3：从数据到结论

```
我收集了以下市场数据：[贴]

帮我：
1. 验证这些数据的可靠性（哪些是官方 / 哪些是推测）
2. 找出数据之间的"矛盾"（如果有）
3. 从这些数据能得出什么结论？（列 3-5 条）
4. 这些结论对我的决策意味着什么？
5. 还缺什么数据才能更有信心？

不要给"看起来对但实际空洞"的结论。
每条结论必须有具体数据支撑。
```

### Prompt 4：国内 + 海外双视角

```
我要调研 [主题]，需要国内 + 海外双视角。

国内部分：
- 中文圈内的讨论（知乎 / 公众号 / 微博）
- 国内主要玩家
- 国内政策 / 合规影响
- 国内用户习惯差异

海外部分：
- 英文圈的讨论（Twitter / Reddit / HN）
- 海外主要玩家
- 海外市场数据
- 中外差异对比

最后：如果我在中国做这件事，和在海外做，分别的机会 / 挑战是什么？
```

## 真实使用 tips

### 1. Gemini Deep Research 是被低估的杀手锏

很多人没用过 Deep Research。**它彻底改变"调研"这件事**：
- 描述主题 → AI 自己跑 30-60 分钟 → 出综述
- 引用 50+ 网页
- 比人手工 1-2 天的调研深度还高

**我自己用 Deep Research 写过几篇 web3-insider 文章** —— 省的不是写作时间，是"读 100 篇文章再下笔"的时间。

### 2. 国内场景用 Kimi 探索版 + 秘塔

中文场景（查公司 / 查行业 / 追新闻）：
- **Kimi 探索版**：中文搜索 + 长文档 + 多轮追问
- **秘塔**：学术 + 中文互联网覆盖广

**两个配合用**：秘塔快搜 → Kimi 深挖。

### 3. 数据可靠性分级

AI 给的数据**不一定对**。你要分级：
- **A 级**：官方财报 / 政府统计 / 上市公司公告
- **B 级**：第三方研究机构（IDC / Gartner / 艾瑞）
- **C 级**：媒体报道 / 行业博客
- **D 级**：社交媒体 / 论坛 / AI 推测

**只有 A/B 级数据能放进正式报告**。C/D 级只做参考。

### 4. "追问 5-10 轮"比"一次问完"好

第一轮答案通常浅。**值钱的是追问**：
- "上面第 3 点的依据是哪条新闻？给我链接"
- "这个公司 2024 年有没有融资，谁投的"
- "把刚才提到的所有数字做成表格"
- "这个数据和另一个来源矛盾，你怎么看"

**追问 5-10 轮后，你会得到一份比手工搜索好 3 倍的资料**。

### 5. 调研结果要"落地"

调研不是目的，**决策才是**。
调研完必须回答：
- **So what?**（这些信息意味着什么）
- **Now what?**（我下一步该做什么）
- **What if?**（如果我的假设错了呢）

让 AI 帮你从"信息"到"结论"到"行动"：
```
基于以上调研，给我 3 个"行动建议"：
- 每个建议：做什么 + 为什么 + 风险 + 时间
- 标出"最推荐"的 1 个 + 理由
```

### 6. 调研沉淀到 Obsidian / Notion

调研结果**不要只存在对话里**：
- 导出到 Obsidian / Notion
- 标日期 + 来源
- 下次做类似调研时复用

**调研是长期资产，不是一次性消耗**。

## 真实案例

我调研"2026 年 AI 工具市场格局"（为 ai-stack 项目）：
- Gemini Deep Research 跑 1 小时（海外英文部分）
- Kimi 探索版追问 10 轮（国内中文部分）
- Claude 帮我写对比矩阵 + 结论
- **总 3 小时出完整调研报告**
- 手做估计 1-2 周

---

## 这套方案的边界

⚠️ 什么时候不适用：

- **极机密 / 非公开信息**：AI 只能搜公开资料，内部数据要人脉
- **极实时**（今天发生的事）：AI 搜索有延迟，突发新闻用 X / 微博
- **极小众 / 极垂直**：AI 训练数据可能没覆盖，要行业人脉
- **需要"一手数据"**：用户访谈 / 问卷 / 实地考察 AI 替代不了
