# 🆕 学一个新领域

> 程序员每年都要"学新东西"，创业者要快速理解新行业，学生要跨学科。**AI 让"花一周学新框架"变成"一天就能动手"**。

---

## 你为什么读到这里?(痛点)

- **不知道从哪开始**：Google 第一页 50% 是 SEO 农场和过期内容
- **信息过载**：100 篇 tutorial 各说各话，不知道选哪个
- **学了忘**：今天看 React，明天看 Vue，后天看 Solid，最后 3 个都模糊
- **不知道"学到什么程度算够"**：永远觉得"还没学完"
- **学习路径不清晰**：没有"从 A 到 B 的最短路径"

---

## 推荐方案

### 🥇 方案 A：AI 给路线图 + Deep Research 补资料 + NotebookLM 消化

**适合**：大主题（新行业 / 新技术栈 / 新学科）
**时间**：4-8 小时（分 2-3 天）
**工具**：[Claude](../tools/foundation-models/claude.md) / [DeepSeek](../tools/foundation-models/deepseek.md) + [Gemini Deep Research](../tools/foundation-models/gemini.md) + [NotebookLM](../tools/research/notebook-lm.md) + [Obsidian](../tools/notes/obsidian.md)

**步骤**：
1. **Day 1 上午**：让 AI 给"骨架 + 学习路径"（Prompt 1）
2. **Day 1 下午**：Gemini Deep Research / Kimi 探索版搜资料
3. **Day 1 晚**：NotebookLM 上传资料 → 听 AI 播客版（通勤 / 散步）
4. **Day 2**：难点 deep dive（对话式追问）
5. **Day 2 晚**：写笔记给"未来的你"（Obsidian）
6. **Day 3**：动手做一个最小项目验证

### 🥈 方案 B：Kimi 探索版 + 秘塔（国内 + 中文主题）

**适合**：国内行业 / 中文资料为主
**时间**：2-4 小时
**工具**：[Kimi 探索版](../tools/foundation-models/kimi.md) + [秘塔](../tools/research/mita.md)
**预算**：¥0

### 🥉 方案 C：直接跟 AI 对话学（最快但最浅）

**适合**：只想"30 分钟了解大概"
**时间**：30-60 分钟
**工具**：Claude / DeepSeek / GPT

## 关键 Prompt 模板

### Prompt 1：开题（进入新领域第一个 prompt）

```
我要学 [领域 / 框架 / 技术 / 行业]。
我现有背景：[已有什么知识]。
学习目的：[做项目 / 调研 / 教学 / 兴趣 / 转行]。
我能投入的时间：[1 天 / 1 周 / 1 个月]。

帮我给一份"学习路线图"：

1. 这个领域的"骨架"（核心 3-5 个概念，用一句话解释每个）
2. 学习路径（从我现有 → 能动手做）的 5-7 个阶段，每阶段：
   - 学什么（具体）
   - 用什么资源（具体到文档 / 视频 / 项目 URL）
   - 学完应该能做什么（可验证的标准）
   - 预计时间
3. 常见的"学习陷阱"（很多人在哪里浪费时间）
4. 3 个判断"我学到了"的具体标准
5. 一个"最小可行项目"（学完后做这个验证自己）

不要"先学基础"这种废话，要给我可执行的具体路径。
```

### Prompt 2：让 AI 当"私人教练"

```
我正在学 [主题]。我的当前理解是：[复述你目前的理解]。

你扮演一个耐心但严格的老师：
1. 先判断我的理解哪里对、哪里错、哪里模糊
2. 对错的地方：纠正 + 解释为什么错
3. 对模糊的地方：用一个简单类比让我"啊哈"
4. 给我一个"下一步该学什么"的建议
5. 给我一个小练习（5 分钟能做完）验证我是否真懂

不要一次给太多信息。一次只教一个概念。
```

### Prompt 3：跨领域类比（最快理解新概念）

```
我懂 [我已经懂的领域]。
我想理解 [新领域的某个概念]。

帮我用 [我已经懂的领域] 的概念做类比：
- [新概念] 类似于 [旧领域的什么]
- 它们的相似点是什么
- 它们的关键差异是什么
- 这个类比在哪里会"崩"（不能过度类比的地方）

用我的语言（[程序员 / PM / 设计师 / 学生]）解释。
```

### Prompt 4：学完后写"给未来的你"的笔记

```
我刚学完 [主题]。这是我的当前理解：[复述]

帮我写一份"半年后我自己再看也明白"的简明笔记（< 800 字）：

结构：
- 一句话核心（这个领域 / 技术 / 概念的本质）
- 必须记的 3-5 个概念（每个 2-3 句话）
- 易混点（容易搞混的 2-3 对概念）
- 实战 tips（3-5 条"官方文档不会告诉你"的经验）
- 不该用的场景（什么时候这个东西不适合）
- 下次该补（我还应该深入哪些方面）
```

### Prompt 5：设计"最小可行项目"

```
我刚学了 [主题]，想做一个小项目验证自己。

帮我设计一个"最小可行项目"：
- 用到我刚学的核心概念（至少 3 个）
- 2-4 小时能做完
- 做完后我能说"我会用 [这个技术] 了"
- 有明确的"完成标准"（什么算做完）
- 给我 step-by-step 的实现路径

不要太简单（hello world 不算）也不要太复杂（做不完）。
```

## 真实使用 tips

### 1. 先"画骨架"再填肉

**不要从某个 tutorial 第一行开始读**。
先用 Prompt 1 让 AI 给你"骨架 + 路径"：
- 你看到全局
- 知道每一步该做什么
- 不会"读到一半发现读错了"

### 2. NotebookLM AI 播客是学习的"隐藏武器"

```
1. 收集 5-10 篇相关资料
2. 上传到 NotebookLM
3. 生成 Audio Overview（AI 播客）
4. 通勤 / 散步 / 跑步时听
```

**听一遍 ≈ 读一遍精华版，但你不用专心坐着**。
详见 [→ workflows/paper-digesting-pipeline](../workflows/paper-digesting-pipeline.md)。

### 3. "学到能教别人" = 真学会了

费曼学习法 + AI：
```
> 我要给一个完全不懂 [主题] 的人解释 [概念]。
> 帮我检查我的解释：[你的解释]
> 哪里不够清楚？哪里有错？怎么改更好？
```

**如果你能教 AI "你的理解"且 AI 说"对"** → 你真懂了。

### 4. 不要一次学太多

人能吸收的"新知识"有上限：
- 同时学 Rust + Wasm + React 19 = 全部变模糊
- 每次专注 1-2 个，**学到能"实战交付一个东西"再切下一个**

AI 工具效率高 → **诱惑你贪多** → 适得其反。

### 5. 写笔记 = 学习的"复利"

最大浪费：
- 花 1 周学了一个东西
- 3 个月后忘 70%
- 半年后又要重新学

每次学完用 Prompt 4 让 AI 帮你出一份 < 800 字的笔记。
存到 [Obsidian](../tools/notes/obsidian.md)。
**未来的你会感谢现在的你**。

### 6. "动手做"比"看 10 篇教程"有效 10 倍

学完理论 → **立刻做 Prompt 5 的最小项目**。
- 做的过程中会发现"我以为懂了其实没懂"
- 卡住的地方 = 真正需要深入的地方
- 做完 = 真学会了

**不要"看完所有教程再动手"** —— 永远看不完。

## 真实案例

我自己 2026-04 学 Hyperliquid 的 HyperBFT 共识：
- Prompt 1 给"骨架"（15 分钟）：知道是基于 HotStuff 改的
- Gemini Deep Research（45 分钟）：出 8000 字综述
- NotebookLM 听播客（20 分钟通勤）
- Prompt 2 追问难点（30 分钟）
- Prompt 4 写笔记（20 分钟）
- **总 < 2.5 小时，达到能写一篇技术深拆文章的水平**
- vs 手工估计 1 整周

## 这套方案的边界

⚠️ 什么时候不适用：

- **真前沿（还没出 paper / 文档）**：AI 不知道
- **需要"动手手感"的技能**（调试直觉 / 设计审美 / 演讲能力）：AI 给不了，要你自己练
- **极小众语言 / 框架**：AI 训练数据少，可能给错
- **领域极其陌生**：背景都不懂，AI 解释你也理解不了 — 先找一个"入门级"的中间跳板
