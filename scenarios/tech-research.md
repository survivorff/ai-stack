# 🔬 技术调研 / 学一个新领域

> 程序员每年都要"学新东西"。AI 让"花一周学新框架"变成"一天就能写代码"。

---

## 你为什么读到这里?(痛点)

- **搜索泥沙俱下**:Google 第一页 50% 是 SEO 农场和过期内容
- **论文 / 官方文档读不动**:啃 RFC / Spec / Paper 慢得让人放弃
- **上手陡峭**:每个框架都觉得"前 3 天像无头苍蝇"
- **看了很多但记不住**:今天看 React,明天看 Vue,后天看 Solid,最后 3 个都模糊
- **不知道业内最佳实践**:看到 100 个 tutorial 各说各话,不知道选哪个

---

## 推荐方案

### 🥇 方案 A:Gemini Deep Research(深度调研)

**适合**: 大主题调研 / 写综述 / 要给团队 / 老板出报告
**时间**: 1-2 小时(含 AI 自动跑 30-60 分钟)
**工具**: [Gemini 3 Pro](../tools/foundation-models/gemini.md) Deep Research
**预算**: $20/月

**步骤**:
1. 描述调研主题 + 你想搞清楚的具体问题
2. Gemini 给你"研究计划" → 你确认 / 改
3. 它跑 30-60 分钟(读 50+ 网页,做笔记,交叉对比)
4. 输出 5000-15000 字带引用的综述

### 🥈 方案 B:Kimi 探索版(国内首选)

**适合**: 国内场景 / 中文资料为主 / 即时调研
**工具**: [Kimi K2.5 探索版](../tools/foundation-models/kimi.md)
**预算**: ¥0(免费)

**步骤**:
1. 探索版打开
2. 直接问主题
3. **追问 5-10 轮** 把表面到深度问出来

### 🥉 方案 C:Perplexity / 秘塔(快速搜索式)

**适合**: 已经知道在找什么,只想快速结果
**工具**: [Perplexity](../tools/research/perplexity.md) / 秘塔
**预算**: 免费档够用

适合"3 分钟搞清一个具体问题",不是大主题深调研。

## 关键 Prompt 模板

### Prompt 1:开题(进入新领域第一个 prompt)

```
我要学 [领域 / 框架 / 技术]。
我现有背景:[已有什么知识]。
学习目的:[做项目 / 调研 / 教学 / 兴趣]。

帮我给一份"学习路线图":

1. 这个领域的"骨架"(核心 3-5 个概念)
2. 学习路径(从我现有 → 能动手做)的 5-7 个阶段,每阶段:
   - 学什么(具体)
   - 用什么资源(具体到文档 / 视频 / 项目)
   - 学完应该能做什么(可验证的标准)
3. 常见的"学习陷阱"(很多人在哪里浪费时间)
4. 3 个判断"我学到了"的具体标准

不要"先学基础"这种废话,要给我可执行的具体路径。
```

### Prompt 2:Deep Research 描述

```
我要做关于 [主题] 的深度调研。

具体想搞清楚:
- 问题 1:[具体]
- 问题 2:[具体]
- 问题 3:[具体]

预期输出格式:
- 5000-10000 字综述
- 核心结论先行(TL;DR)
- 主要章节(按问题组织)
- 每个论断都有引用源
- 标出"业内仍有争议的"地方
- 最后给"行动建议"

读者:[谁读 — 工程师 / 老板 / 自己 / 团队]
```

### Prompt 3:对比 N 个方案

```
我要在 X 和 Y 和 Z 之间做选择,这是个 [类型] 决策。

帮我做对比:

| 维度 | X | Y | Z |
|------|---|---|---|

我关心的维度(请基于这些扩):
- 学习曲线
- 性能
- 生态
- 中长期维护
- 团队招聘难度
- [我特定关心的]

每格简短说明 + 引用源(最近 6 个月内的为主)。

最后给:在我场景下的推荐 + 1 句话理由。
我场景:[描述]。
```

### Prompt 4:啃论文 / Spec / RFC

```
这是一篇 paper / spec / RFC:[贴 PDF / link]

帮我:

1. 一句话定位:这篇核心想说什么?
2. 主要章节摘要(每节 2-3 句话)
3. 它解决的问题在工业界的现状是什么?
4. 它的创新点(列 3-5 条具体)
5. 它的不足 / 局限(作者承认的 + 你能看出的)
6. 我应该看哪 3 个 figure / 表格 才能"懂大意"
7. 我应该读哪 1-2 节 才能看到细节

让我用 30 分钟达到"60% 理解度",剩下的我自己 deep dive。
```

### Prompt 5:学了后写"我的笔记"

```
我刚学完 [主题]。这是我的当前理解:[复述自己的理解]

帮我:

1. 找出我理解错的地方
2. 找出我没意识到的"经验之谈"(不会写在 tutorial 里但实际很重要)
3. 找出我没看到的"陷阱"
4. 给我一份"半年后我自己再看也明白"的简明笔记(<800 字)

笔记结构:
- 一句话核心
- 必须记的 3 个概念
- 易混点
- 实战 tips(3-5 条)
- 不该用的场景
```

**这一步的价值**:**让"看了忘"变成"沉淀下来的知识"**。

## 真实使用 tips

### 1. Gemini Deep Research 是被低估的杀手锏

很多人没用过 Deep Research,**它彻底改变"调研"这件事**:
- 描述主题 → AI 自己跑 30-60 分钟 → 出综述
- 引用 50+ 网页
- 比人手工 1-2 天的调研深度还高

我自己用 Deep Research 写过几篇 [web3-insider](https://github.com/survivorff/web3-insider) 文章 —— **省的不是写作时间,是"读 100 篇文章再下笔"的时间**。

### 2. 国内场景用 Kimi 探索版

中文场景(查公司 / 查行业 / 追新闻):**Kimi 探索版 vs Gemini Deep Research 国内方面更强**(中文资料覆盖度高)。
- 国产模型:**Kimi 探索版默认**
- 海外英文 / 学术:**Gemini Deep Research**

### 3. 学新东西先"画骨架"再填肉

**不要从某个 tutorial 第一行开始读**。
先用 Prompt 1 让 AI 给你**"骨架 + 路径"**:
- 你看到全局
- 知道每一步该做什么
- 不会"读到一半发现读错了"

### 4. 对比方案别只看"哪个最好"

3 个方案给你:**别选"最好的"**,**选"和你场景最匹配的"**:
- 团队都没用过 X,哪怕 X 客观最好,你选 Y
- 公司其他系统都用 Y,哪怕 X 略好,你选 Y
- 你 6 个月会离职,哪怕 X 好一点,你选 Y

让 AI 在 Prompt 3 强调"**我场景**" → AI 给的推荐才贴你。

### 5. 啃论文优先看 Figure 和 Conclusion

任何 paper:
- **Abstract**(摘要,1 分钟)
- **Conclusion**(结论,2 分钟)
- **Figure 1 + Figure 最大那张**(5 分钟)
- 看完上面 8 分钟后,**90% paper 你已经知道值不值得继续读**

让 AI 帮你**优先理解 Figure**(Prompt 4 里强调)。

### 6. 学完一定写笔记给"未来的自己"

最大浪费:
- 花 1 周学了一个东西
- 3 个月后忘 70%
- 半年后又要重新学

每次学完用 Prompt 5 让 AI 帮你出一份 < 800 字的笔记。
**未来的你会感谢现在的你**。

### 7. 别一次学太多东西

人能吸收的"新知识"有上限。
- 同时学 Rust + Wasm + React 19 = 全部变模糊
- 每次专注 1-2 个,**学到能"实战交付一个东西" 再切下一个**

AI 工具效率高 → **诱惑你贪多** → 适得其反。

## 真实案例

我自己 2026-04 学 Hyperliquid 的 HyperBFT 共识:

- Step 1:Prompt 1 给"骨架"(15 分钟)
  - 知道是基于 HotStuff 改的
  - 知道关键差异点
- Step 2:Gemini Deep Research(45 分钟)
  - 出了 8000 字综述
  - 引用 30+ 资料(论文 / 博客 / 官方 docs)
- Step 3:Prompt 4 啃 paper(30 分钟)
- Step 4:Prompt 5 写笔记(20 分钟)
- 总耗时 < 2 小时,达到能写一篇技术深拆文章的水平

**vs 手工估计 1 整周**。

---

## 这套方案的边界

- **真前沿(还没出 paper / 文档)**:AI 不知道
- **公司机密技术**:AI 训练数据里没有
- **极小众语言 / 框架**:AI 训练数据少,可能给错
- **需要"动手手感"的技能**(比如调试 / debug 直觉):AI 给不了,要你自己练
