# 飞书智能伙伴

> 飞书的 AI 全家桶。**国内团队 OA + AI 的 default**。
> Tier A。**类似 Notion AI 的国内版,但更面向企业 / 协作**。

---

## 基本信息

| 项 | 值 |
|---|---|
| **厂商** | [飞书 / 字节跳动](https://www.feishu.cn) |
| **形态** | 飞书内嵌(IM / 文档 / 表格 / 视频会议 / 任务 全平台) |
| **底层模型** | 字节豆包 + Claude / GPT(部分功能可选) |
| **国内可用性** | 🟢 完全直连 |

---

## Tier 评级:A

**评级理由**:
- **国内团队 default**:中国大陆中小团队 / 创业公司用飞书的比例第一
- **集成最深**:IM 群消息 / 文档 / 表格 / 日历 / 会议 / 任务 全部 AI
- **跨工具搜索**:类 Notion Connections,但**飞书原生闭环更顺**
- **中文母语 + 国内可用 + 移动端体验好**

**不是 A+ 的原因**:
- **vendor lock-in 强**(切换成本高)
- **顶级模型对话**仍不如直接用 Claude / GPT
- **出海** 适配差(主要面向中国大陆)
- **个人 / 单人** 用没意义(必须团队场景)

---

## 它最适合做什么

### ✅ 国内团队的"AI OA 工作流"

如果你团队已经在飞书:
- **群消息 @飞书**:即时 AI 答
- **飞书文档**:选中文字 → AI 改写 / 总结 / 翻译 / 续写
- **视频会议**:自动纪要 + Action Items
- **跨群搜**:"上周讨论 X 的对话在哪"
- **多维表格**:AI 自动填充 / 分析

**国内打工人不开等于浪费**(本身就在飞书里)。

适合:
- [会议纪要](../../scenarios/meeting-notes.md)
- [工作汇报 / 周报](../../scenarios/work-reporting.md)
- [写 PRD](../../scenarios/prd-writing.md)
- [用户反馈整理](../../scenarios/feedback-synthesis.md)

### ✅ 飞书会议自动纪要

- 会议进行中自动转写
- 会议结束 → AI 摘要 + Action Items + 关键决策
- 自动通知相关人

**对管理层 / PM:省 80% "整理纪要"时间**。

### ✅ 跨群 / 跨日历 / 跨文档搜

最值钱的功能:
> "找出 2 月份关于支付那次讨论的核心结论"

飞书 AI 跨所有群消息 / 日历 / 文档搜 + 综合答。
**比 Slack + Notion 跨工具搜更顺**(因为飞书是自家闭环)。

### ✅ 中小团队 / 创业公司

10-200 人团队场景最优:
- 飞书免费档(50 人以下)很大方
- AI 集成包含在订阅里
- 启动成本低 + 协作顺

## 它**不擅长**什么

### ❌ 你不在飞书

这是最关键的:**不用飞书就没意义**。
个人用户 / 不用飞书的团队 → 没必要了解。

### ❌ 顶级模型对话

飞书 AI 默认走豆包 / Claude / GPT 路由,**复杂任务 vs 直接用 Claude / GPT 网页略弱**。
严肃写作 / 复杂推理:**仍切回主力模型**。

### ❌ 出海

飞书国际版 Lark 海外用户少,集成深度不如国内。
出海团队 → Slack / Notion / Asana 国际化方案。

### ❌ 隐私敏感

飞书数据在字节体系内。
极敏感场景 → Obsidian 自部署 / 自部署知识库。

## 真实使用 tips

### 1. @飞书 / @智能伙伴 在群里

任何群里 @飞书:
- "总结一下今天的讨论"
- "找一下上周关于 X 的消息"
- "@张三 的负责事项是什么"

**省的不是发消息时间,是"翻历史"时间**。

### 2. 文档 AI 能力深度

飞书文档 AI:
- 选中段落 → 改写 / 摘要 / 翻译
- "AI 帮我写"长文(类 Notion AI Continue)
- 文档插入图表(基于内容自动)

工作流:**飞书文档 + AI ≈ Notion + AI 国内版**。

### 3. 视频会议自动纪要质量稳

飞书视频会议(妙记):
- 实时转写
- 关键决策标记
- Action Items 提取
- 自动同步到任务

**省 PM 一个 1h 会议 30min 的整理时间**。

### 4. 多维表格 + AI

飞书多维表格 + AI:
- 自动从文本提取结构化字段
- 自动分类 / 打标签
- 自动汇总 / 报表

**类 Notion Database + AI,但中文场景体验更顺**。

### 5. 自定义"机器人 / 智能助手"

飞书开放平台可以:
- 给团队定制 AI 助手
- 接公司知识库 RAG
- 接公司业务系统

**类似 GPTs 但内嵌飞书 + 私有部署**。

### 6. 国内团队工作流推荐

```
飞书智能伙伴(团队协作 / 会议 / 跨群)
+ 通义灵码(开发者 IDE 补全)
+ Cursor / Claude Code(开发者主力)
+ DeepSeek / Kimi(通用问答 / 调研)
+ 即梦 / Suno / Speech-02(创作)
```

**国内团队 AI 配置基本就这套**。

## 一段判断

国内团队场景:
- **大公司 / 国企** → 钉钉(阿里生态)/ 企业微信
- **互联网中小团队 / 创业公司** → **飞书**(产品最现代)
- **海外团队** → Notion + Slack + Linear

飞书智能伙伴**对"已经在飞书"的团队**:**几乎是免费送的产能升级**。
**不在飞书 → 没必要专门关注**。
