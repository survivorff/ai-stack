# 🤝 贡献指南

> 这是一个**有立场的 Tier List**,不是大杂烩。所以贡献门槛比一般 awesome list 高。

---

## 三个原则

### 1. 必须真实用过

PR 加新工具必须有"我用了它做什么"的具体说明。例如:

> ✅ 好:"我用 Cursor 写了我现在用的博客系统(Astro + TypeScript),3 个月时间,Composer 帮我重构了 8 次架构"
> ❌ 不好:"Cursor 是基于 VS Code 的 AI 编辑器,支持多种模型..."(官方文案)

### 2. 必须标国内可用性

- 🟢 **直接可用**:国内网络可用,中国信用卡可付
- 🟡 **需要自配**:需要代理 / 海外卡 / API key 配置
- 🔴 **不可用**:国内基本无法用

### 3. 必须有"它不擅长什么"

每个工具笔记的最重要字段。这是**这份列表区别于营销文章的核心**。

---

## 怎么加新工具

### 方法 A:开 Issue 推荐(轻量)

如果你不想自己写,开 Issue 告诉我:

```
工具名:
分类:
我用过它做的具体事(必填):
最值钱的用法:
不擅长什么:
国内可用性:
价格:
```

我每月 review 一次 issue。值得加的会进 changelog。

### 方法 B:开 PR(深度)

复制 [`_template.md`](./categories/_template.md) 写一份完整笔记。
PR 标题格式:`[Add] <分类>/<工具名>`

我会 24-72 小时内 review。

---

## 怎么改 Tier 评级

不接受"改 tier"的 PR。Tier 是**我个人观点**。

但你可以开 Issue 表达不同意,我会在月度 review 时考虑。

---

## 不接受的贡献

- ❌ 工具厂商自我推销(不管包装多隐蔽都拒)
- ❌ 复制官方文档作为工具描述
- ❌ 加未公开/未上线的工具(必须公开可用)
- ❌ 加 3 个月没更新的"已死项目"
- ❌ 加未通过基础合规审查的工具(数据隐私问题大的)
- ❌ 攻击其他工具的内容(可以诚实评价缺点,但不撕)

---

## 我会加什么样的新工具

优先级排序:

1. **打破现有 Tier S**(说明出现颠覆性新东西,要立刻评估)
2. **国产替代海外**(中国开发者更需要这种)
3. **垂直场景颠覆**(在某个具体场景把现有工具甩开 5x)
4. **价格颠覆**(同等效果一半价格)
